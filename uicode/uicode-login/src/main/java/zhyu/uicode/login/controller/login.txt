登陆的场景非常多,也和安全服务,第三方应用,本方多个应用联系紧密.
1.对于一个应用,一个服务器 那么登陆的方式就在授认证上下功夫.
	普通的有 session 管理; token 管理,框架如spring-security;shiro等;
	自己手写可以在控制层(mvc多层架构)上加 处理逻辑(filter;拦截器;总控制器;aop的方法前环绕);
2.认证方式:
	PKI/CA数字证书认证;IP认证;时间段认证;访问次数认证;各认证方式如果模块化结合的化可以方便装卸;
	--之后便有了认证组合
3.对于大型系统,认证和授权会用到多个服务器(单点登陆),比如会可能会用到缓存服务器,LDAP服务器,甚至还有专门的管理员认证授权服务器
